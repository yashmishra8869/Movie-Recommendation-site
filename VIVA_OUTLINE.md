# B.Tech Viva Outline: Hybrid Movie Recommendation System

## Slide 1: Title and Team
- Project: Hybrid Movie Recommendation System
- Domain: Machine Learning + Full Stack Development
- Stack: Python, Flask, Scikit-learn, Tailwind CSS, Vercel

## Slide 2: Problem Statement
- Movie platforms have massive catalogs and user choice overload.
- Pure content-based systems miss user behavior patterns.
- Pure collaborative systems struggle with cold-start users/items.

## Slide 3: Objective
- Build a deployable recommender web app with:
  - Content-based intelligence
  - Collaborative personalization
  - Hybrid weighted fusion
- Deliver practical UI with real poster metadata.

## Slide 4: System Architecture
- Frontend: Tailwind dark-themed UI with autocomplete search.
- Backend: Flask API in api/index.py (Vercel compatible).
- Data Layer: Sample dataset (50 movies) generated by data_loader.py.
- External API: TMDB for poster images.

## Slide 5: Dataset Design
- Fields: title, genres, overview, tmdb_id.
- Generated sample dataset for immediate demonstration.
- No external large datasets required for basic run.

## Slide 6: Content-Based Module
- Combine genres + overview as text corpus.
- TF-IDF vectorization converts text to numeric features.
- Cosine similarity computes closeness between movie vectors.
- Output: content_score per candidate movie.

## Slide 7: Collaborative Module (Simulated SVD-like)
- Generate a deterministic default user latent vector for demo personalization.
- Dot product with movie latent factors gives affinity score.
- Normalize scores to [0,1] for stable blending.
- Output: collaborative_score per candidate movie.

## Slide 8: Hybrid Fusion Logic
- Formula:
  - hybrid_score = 0.7 x content_score + 0.3 x collaborative_score
- Ranking based on descending hybrid_score.
- Returns top-N recommendations excluding input movie.

## Slide 9: Simplified Personalization UX
- User does not enter user_id in the interface.
- Backend uses an internal default profile for consistent recommendations.
- Keeps UI simple for project demonstration.

## Slide 10: API Design
- GET /: serves UI template.
- GET /api/autocomplete?q=: search suggestions.
- POST /api/recommend: hybrid recommendation response.
- GET /health: status and loaded movie count.

## Slide 11: Frontend Experience
- Netflix-style dark UI for visual appeal.
- Search with autocomplete-like behavior.
- Responsive cards with poster, genre, overview, and score.
- Mode label indicates hybrid vs cold-start content-only.

## Slide 12: Deployment
- Vercel Python runtime using vercel.json.
- Entrypoint configured as api/index.py.
- TMDB key provided via environment variable.

## Slide 13: Testing and Validation
- Syntax validation using py_compile.
- API smoke tests:
  - /health
  - /api/autocomplete
  - /api/recommend
- Manual UI test for end-to-end flow.

## Slide 14: Limitations and Future Work
- Current collaborative part is simulated, not trained on real user logs.
- Small demo dataset limits recommendation diversity.
- Future upgrades:
  - Real user-item matrix factorization
  - Feedback loop and ratings collection
  - Model evaluation metrics (Precision@K, Recall@K, NDCG)

## Slide 15: Conclusion
- Successfully built a full-stack, deployable hybrid recommender.
- Solves cold-start gracefully and supports personalized output.
- Project demonstrates ML concepts + production deployment readiness.
